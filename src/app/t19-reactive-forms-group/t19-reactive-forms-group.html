Reactive FormGroup in Angular:

    1. What is FormGroup:
      • FormGroup is a collection of FormControl objects.
      • It represents the entire form in reactive forms.
      • FormGroup tracks value and validation state of the form.
      • It is defined in the component TypeScript file.

    2. Why we use FormGroup:
      • To manage multiple form controls together.
      • To apply validations at form level.
      • To handle complex and dynamic forms.
      • To get full control over form data and state.

    3. How FormGroup Works:
      • FormGroup contains one or more FormControl.
      • Each FormControl represents an input field.
      • FormGroup is bound to HTML using formGroup directive.
      • Inputs are connected using formControlName.

    4. Steps to Implement Reactive FormGroup:
      1. Module Configuration
        • Import ReactiveFormsModule.
      2. Component TS
        • Import FormGroup and FormControl.
        • Create FormGroup object.
        • Define FormControl for each input field.
      3. HTML File
        • Bind form using formGroup directive.
        • Use formControlName to connect inputs.

    5. Example FormGroup Structure:
      • FormGroup
        • name → FormControl
        • email → FormControl
        • password → FormControl

    6. Advantages of FormGroup:
      • Better control over form values and validation.
      • Easy to add or remove form controls dynamically.
      • Improves form testing and maintainability.
      • Suitable for large and complex forms.

    7. One Line Summary:
      • FormGroup is used in reactive forms to manage and validate multiple form controls as a single unit.

Reactive Form Validations Used in Application:

    1. What are Form Validations:
      • Form validations are rules applied to input fields.
      • They prevent invalid or empty data submission.
      • In reactive forms, validations are defined in TypeScript.

    2. Validators Used in This Form:
      • required
      • maxlength
      • email
      • minlength

    3. Required Validator:
      • Ensures the input field is not left empty.
      • Used for name, email, and password fields.
      • Error message is shown when the field is touched and empty.

    4. MaxLength Validator:
      • Limits the maximum number of characters allowed.
      • Used for the name field.
      • Prevents very long or invalid names.

    5. Email Validator:
      • Checks whether the entered value is a valid email format.
      • Used only for the email field.
      • Prevents incorrect email addresses.

    6. MinLength Validator:
      • Ensures minimum required characters are entered.
      • Used for the password field.
      • Improves password security.

    7. Error Message Display Logic:
      • Error messages are displayed using *ngIf.
      • Errors are shown only when the field is touched.
      • Uses the errors property of FormControl.

    8. Submit Button Validation:
      • Submit button is disabled when the form is invalid.
      • Uses profileForm.invalid property.
      • Prevents submission of invalid form data.

    9. Set Default Value Validation:
      • setValue() method assigns values to all form controls.
      • All controls must receive a value.
      • If any control is missing, Angular throws an error.

    10. Advantages of Reactive Form Validations:
      • Centralized validation logic.
      • Easy to manage and test.
      • Provides better user experience.
      • Suitable for real-world applications.

    11. One Line Summary:
      • Reactive form validations ensure only valid and secure data is submitted by applying rules to form controls.
