Dynamic Routing in Angular:

    1. What is Dynamic Routing:
      • Dynamic routing is used to pass values through the URL.
      • It allows loading different data using the same component.
      • Dynamic values are passed using route parameters.
      • Example URLs: /users/1, /users/2, /users/3

    2. Why we use Dynamic Routing:
      • To avoid creating multiple components for similar pages.
      • To load data dynamically based on user selection.
      • To pass ID or key through the URL.
      • Used in real-world apps like product details, user profiles.

    3. How Dynamic Routing Works:
      • Route parameters are defined using :id.
      • Data is fetched using ActivatedRoute.
      • Navigation is done using routerLink with parameters.

    4. Steps to Implement Dynamic Routing:
      1. Route Configuration
        • Define dynamic parameter in routes using :id.
      2. HTML (List Page)
        • Create dynamic links using routerLink.
        • Pass ID in the URL.
      3. Component TS (Details Page)
        • Import ActivatedRoute.
        • Read route parameter using paramMap.
        • Load data based on received ID.

    5. Example URL Flow:
      • /t18 → Displays Users List
      • /t18/1 → Displays User Details
      • /t18/5 → Displays Another User Details

    6. Advantages of Dynamic Routing:
      • Reduces number of components.
      • Makes application scalable.
      • Improves URL readability.
      • Easy navigation and bookmarking.

    7. One Line Summary:
      • Dynamic routing allows Angular applications to load data dynamically using URL parameters.
